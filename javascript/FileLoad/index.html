FileLoad

Faz a leitura de arquivos para manipulação desses ainda na máquina do usuário.

Utiliza recuros da API FileReader

Apresentação

onStart

onError

onProgress

onResult

Processo de leitura terminado

Consulte o log no console

Erro de leitura

Consulte o log no console

Selecione apenas aquivos menores que 200mb's

Resultados serão exibidos no console do navegador

Utilização

Adicione o arquivo necessário.

<script src="FileLoad.js"></script>

Declare um objeto para FileLoad ao final do documento.

<script>
    var ler = new FileLoad();
</script>

Para inicar deve-se chamar o método load especificando como parâmetro o arquivo para ser lido e as opções/callback's.

As opções e callback's é de livre escolha, ou seja pode-se usar ou não, mas o arquivo é necessário.

O arquivo é obtido através de um <input type="file" />.

Exemplo:

<input type="file" id="exemplo" />
<script>
    var ler = new FileLoad(); // Instância do FileLoad
    var input = document.getElementById('exemplo'); // Identificar o input
    var arquivo = null;

    input.addEventListener('change', iniciarLeitura, false); // Quando algo é selecionado

    // Função de execução
    function iniciarLeitura(evento) {
        arquivo = evento.target.files[0]; // Arquivo anexo ao input['file']

        // ler é o nome da instância do FileLoad
        ler.load(arquivo, {readType: 'data'});
    }
</script>

A função load() que consiste em dois parâmetros:

load(A, B)

A) -> Informar um arquivo selecionado pelo usuário.

B) -> Informar a opção e os métodos callback's dentro de um objeto.

Qualquer método é opcional!

Use somente aqueles que achar necessário.

Opção

readType

Deve ser informado como string para dizer como o arquivo deve ser lido.

Os tipos disponíves são:

  • data ⇒ Resultado será um data URL representando os dados do arquivo.
  • binary ⇒ Resultado será os dados binários.
  • text ⇒ Resultado será o conteúdo do arquivo como texto.

NOTA:

Se não informado ou o valor não seja 'data', 'binary' ou 'text' o valor padrão será data

Mesmo que não seja necessário aconselho especificar qual tipo de leitura deve ser feita.

Opções (Métodos/Callback's)

Todos métodos a seguir são opcionais, use se achar necessário

onStart

Deve informado uma função que será executada quando o quando o processo de leitura iniciar.

Você poderá adicionar um parâmetro de retorno, exemplo:

onStart: function (param) {}

Dentro da função param será os dados pré processados do arquivo.

Então nesse método você pode avaliar o arquivo, digamos que somente queira arquivos PDF e o usuário selecinou qualquer outro tipo.

Nesse caso você poderá negar todo processo restante, interrompendo o FileLoad de prosseguir retornando false dentro do escopo do método.

Exemplo:

onStart: function (param) {
    if (param.type !== 'application/pdf') { // Se não for um arquivo PDF
        return (false);  // O FileLoad só executará esse método e vai parar aqui devido ao retorno falso
    }
}
onError

Deve informado uma função que será executada sempre que ocorrer algum erro na leitura do arquivo.

Assim como o método onStart você também pode adicionar algum parâmetro que lhe informará qual erro ocorreu.

onError: function (param) {}

Este é um recurso da API FileReader, mas não consegui de forma alguma simular ou disparar algum erro.

Portanto francamente não sei se de fato vai funcionar quando preciso.

De qualquer forma achei necessário, caso aconteça um eventual erro imprevisto.

onProgress

Deve ser informado uma função que será executada repetitivamente até que o processo de leitura termine ou aconteça algum erro.

Sim! Essa função vai ficar se repetindo, portanto cuidado.

Você também poderá adicionar um parâmetro que lhe informará um objeto com o tamanho original do arquivo e o quanto dele já foi lido.

  • total ⇒ O tamanho em bit's do arquivo.
  • loaded ⇒ O quanto em bit's já foi lido.

Exemplo:

onProgress: function (ev) {
    console.log('Completado: ' + ev.loaded + ' | Tamanho total: ' + ev.total);
}
onResult

Deve ser informado uma função que será executada quando todo o processo de leitura é terminado.

Assim como nos métodos anteriores você adiciona um parâmetro que lhe informará o resultado da leitura, de acordo com o tipo que você definiu em readType.

onResult: function (resultado) {}

Método Extra

Para parar a leitura, basta utilizar a função cancel(), pode ser ativada a qualquer momento para abortar a operação.

Exemplo:

<input type="file" id="exemplo" />
<button id="parar">Abortar Leitura</button>
<script>
    var ler = new FileLoad();
    var arquivo = document.getElementById('exemplo');
    
    document.getElementById('parar').addEventListener('click', function () {
        ler.cancel(); // Para a leitura
    }, false);
    
    ler.load(arquivo.files[0], {readType: 'data'});
</script>

Exemplo Prático

<input type="file" />
<button onclick="lerArquivo()">Ler Arquivo</button>
<script>
    var ler = new FileLoad(); // Instância do FileLoad

    // Identifique o input file
    var input = document.getElementsByTagName('input')[0];

    function lerArquivo() {
        // ler é o nome da instância do FileLoad
        ler.load(input.files[0], {
            readType: 'text',
            onStart: function (arquivo) {
                // O que fazer quando é iniciado
            },
            onError: function (erro) {
                // O que fazer se acontecer erro
            },
            onProgress: function (progresso) {
                // O que fazer enquanto o arquivo está sendo processado
            },
            onResult: function (resultado) {
                // O que fazer quando terminar
            }
        });
    }
</script>

Ponderação

Tenha em mente que o FileLoad não faz upload, todo o processo é executado na máquina do usuário.

Dependendo da capacidade de processamento da máquina do usuário arquivos pequenos/medianos serão lidos quase que instantaneamente.

E arquivos grandes demais, maior que a memória da máquina disponível resultará em OUT_OF_MEMORY.

Então você pode está limitando o tamanho do arquivo no método onStart, enfim fica a seu critério, afinal de contas não dar para saber qual é a capacidade da máquina do usuário, para prever eventuais problemas relacionados a memória.

Mas avalie cenários:

  • Arquivos grandes que podem "travar" a máquina.
  • Log da má execução da sua página enviada para equipe de desenvolvimento do navegador.
  • Bloqueio da sua página devido a danos que o mau uso do script pode causar na máquina do usuário.
