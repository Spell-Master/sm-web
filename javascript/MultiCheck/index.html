MultiCheck.js

O MultiCheck trabalha com multiplas entradas de caixa de checagem.

Para o envio de vários dados simultâneos dos mesmo.

Basicamente seu conceito se aplica como um carrinho de compras, mas sem o apelo visual que tal aplicação deve possuir.

Assim então usando de algo mais simples de forma a uso em demais situações corriqueiras onde o envio múltiplo de dados deve ocorrer.

Apresentação:

Utilização

Adicione os arquivos necessários:

<link href="MultiCheck.css" rel="stylesheet" type="text/css"/>
<script src="MultiCheck.js" type="text/javascript"></script>

É opcional criar uma estrutura HTML, que nada mais é que criar um ou mais input "checkbox", ao qual a interação deve ocorrer assim que o MultiCheck carregar.

É importante que consiga identificar esses input para serem repassados para o MultiCheck 
<input type="checkbox" name="input-a" value="valor a" class="exemplo" />
<input type="checkbox" name="input-b" value="valor b" class="exemplo" />
<input type="checkbox" name="input-c" value="valor c" class="exemplo" />
Perceba o uso do classificador exemplo ao qual usarei para informar ao MultiCheck que ele deve interagir com esses input.

Inicie em um javascript ao final do documento criando um objeto que instância a classe MultiCheck.

Esse objeto poderá ser usado para acessar os demais métodos da classe.

<script>
    var multi = new MultiCheck({
        addInput: document.getElementsByClassName('exemplo'),
        maxCheck: 3,
        btnText: 'Olá mundo',
        btnClass: 'um_classificador_qualquer',
        onAction: 'https://url_do_post/arquivo.php',
        onSubmit: function (e) {
            console.log(e);
        }
    });
</script>

Parâmetros Iniciais

Qualquer parâmetro inicial é opcional, portanto use aqueles que achar de deve ser usado.

  • addInput: Assim como dito anteriormente, você pode informar quais input vão está sobre os cuidados do MultiCheck.

    Cada input deve possuir o atributo name ao qual você irá utilizar no back-end para saber quais dados serão enviados pelo formulário.

    E deve possuir o atributo value ao qual você irá utilizar no back-end para saber quais são os valores que o usuário selecionou.

    Caso não queira o utilização desse parâmetro, você poderá está usando os métodos auxiliares para adicionar um ou mais input ao script posteriomente.

  • maxCheck: Aqui você informa um número que indicará qual a quantidade máxima de input que o usuário pode marcar.

    Se não definido o valor máximo será "99" noventa-e-nove.

  • btnText: Aqui você informa um texto que será mostrado no botão de validação do formulário.

    Se não definido o texto será "Enviar"

  • btnClass: Aqui você informa um classificador ".class" para o botão de envio.

    Use para estilizar conforme queira o botão de envio.

  • onAction: Aqui você informa para onde os dados serão enviados através do protocolo "POST".

    Se não informado os dados serão enviados para a própria url atual.

    Esse parâmetro se torna obsoleto/inutilizável quando o parâmetro "onSubmit" estiver presente.

  • onSubmit: Aqui você informa uma função que será executada substituíndo o comportamento padrão do navegador com o formulário.

    Você poderá adicionar um parâmetro ao escopo da função, esse parâmetro será o formulário de envio. Exemplo:

    var multi = new MultiCheck({
    onSubmit: function (param) {
        console.log(param); // "param" será o formulário criado pelo MultiCheck
    }
});

Métodos extra

  • addInput: Como esse método você pode adicionar novos input aos cuidados do MultiCheck. Exemplo: 
    var multi = new MultiCheck();
multi.addInput(
    document.getElementById('um_input_checkbox')
);
  • addHidden Com esse método você pode criar input extras no formulário para suas informações serem usadas no back-end. Exemplo: 
    var multi = new MultiCheck();
var novo = document.createElement('input');
novo.name = 'alguma_coisa';
novo.value = 'algum valor';
multi.addHidden({
    name: novo.name,
    value: novo.value
});

    Dois prâmetros são obrigatórios para esse método:

    name:Ao qual você deve informar o "atributo name" para o input que será criado no formulário.

    value:Ao qual você deve informar o "atributo value" para o input que será criado no formulário.

    Qualquer input criado dessa forma para o formulário não poderá ser removido
  • clearAll: Com esse método você desmaca e remove todos checkbox selecionados pelo usuário
